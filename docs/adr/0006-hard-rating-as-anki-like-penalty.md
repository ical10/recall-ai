# "Hard" rating as a passing penalty, not a failure

The canonical SM-2 paper treats every `quality < 3` rating as a full failure — `repetitions` resets to 0 and `interval_days` returns to 1. That means a card rated "Hard" (q=2) is indistinguishable from "Again" (q=0) at the schedule level: it never graduates so long as the user keeps finding it slightly effortful, defeating the SRS purpose. We deviate from the paper: `compute_next_review` now branches Hard separately — `repetitions` keeps incrementing, `interval_days = max(1, round(prev * 1.2))`, and `ease_factor` drops by `HARD_EASE_PENALTY` (0.15) floored at `EASE_FLOOR`. Again remains a full failure.

Considered (a) staying literal to SM-2 — rejected because every modern SRS (Anki, RemNote, Mochi) diverges here for the same reason; (b) dropping the Hard button and folding it into Again — rejected because the gradation is useful UX and the rating buttons are already wired in Slice B. The constants `HARD_INTERVAL_MULTIPLIER = 1.2` and `HARD_EASE_PENALTY = 0.15` are starting values matching Anki's defaults; if cards graduate too fast on repeated Hard ratings, the multiplier is the first dial to tune toward 1.0 (no interval growth on Hard).
